I love building websites. And over the years, I've tried a lot of tools to do it well. When I discovered [Astro](https://astro.build), something clicked. It matched exactly how I think about the web — and I've never looked back. In this post I want to share what Astro is, how it works, and why it became my tool of choice for every website I build.

## What even is a framework?

If you're not a developer, the word "framework" might sound abstract. Think of it like this: a framework is a set of building blocks and rules that a developer uses to construct a website. Instead of writing everything from scratch each time, a framework gives you a solid foundation to build on.

There are many frameworks out there. Some are built for complex web *applications* — things like dashboards, social platforms, or tools where everything updates in real time. Others are better suited for websites: pages that present content clearly, load fast, and work well for everyone.

Astro belongs firmly in the second category — and it excels at it.

## What is Astro?

[Astro](https://astro.build) is a framework for building websites. It's open source, actively maintained, and has a wonderful community around it. What makes Astro special is its philosophy: **ship as little as possible to the browser**.

Most modern frameworks send a lot of JavaScript to the visitor's browser. JavaScript is a programming language that runs in the browser and makes things interactive. It's powerful — but it also takes time to download, parse, and execute. For a website that mainly shows content, most of that JavaScript is simply not needed.

Astro takes a different approach. By default, it renders your pages to plain HTML at build time — before anyone even visits the site. When a visitor opens a page, they receive clean, ready-to-read HTML. No waiting for JavaScript to load. No blank screen while the page assembles itself. Just the content, immediately.

This is called **static site generation**, and it's a brilliant match for most websites.

## How Astro works — in plain language

Imagine you're baking bread. You have two options:

1. **Bake it fresh for every customer** as they walk into the shop. Every order takes time. The customer waits.
2. **Bake it in the morning** so it's ready when people arrive. Customers walk in and the bread is already there.

Many traditional web frameworks use the first approach — they generate pages on the fly for every visitor. Astro uses the second: it builds your pages in advance, so every visitor gets an instant response.

In technical terms, Astro compiles your website into static files — HTML, CSS, and a small amount of JavaScript where needed. These files are then served from a hosting platform (I use [Vercel](https://vercel.com)) that delivers them from servers around the world, close to wherever your visitor is. The result is a website that feels instant.

## Why I'm so enthusiastic about Astro

I genuinely enjoy working with Astro every day. Here's what makes it stand out for me.

### The performance is remarkable

Pages built with Astro score exceptionally well on [Google PageSpeed](https://pagespeed.web.dev) and [Core Web Vitals](https://web.dev/vitals/) — the performance metrics that Google uses to evaluate websites. Fast websites rank better in search results, keep visitors engaged, and simply feel better to use.

For my clients — whether they're a freelancer, a local business, or a creative professional — a fast website makes a real difference. People don't wait. A site that loads instantly creates a great first impression.

### It's built for content

Astro has a feature called [Content Collections](https://docs.astro.build/en/guides/content-collections/) — a structured way to manage all the content on a site. Blog posts, portfolio projects, team members, FAQs: everything lives in organised files with clear rules. It keeps things tidy, predictable, and easy to maintain over time.

This blog you're reading right now is powered by Astro's content collections. Every post is a simple file. Adding new content is fast and clean.

### Smart JavaScript where it counts

Not every website is purely static. Sometimes you need a contact form, a search bar, or an interactive component. Astro handles this beautifully with what it calls [Islands Architecture](https://docs.astro.build/en/concepts/islands/).

The idea is simple: the page is mostly static HTML, but certain components — "islands" — can be made interactive. Only those islands load JavaScript. The rest of the page stays lean and fast.

This means I can build websites that are both performant *and* fully functional — without any compromise.

### The developer experience is a joy

I love working in Astro's file format. It feels familiar to anyone who knows HTML, but it's more powerful. You can use components, layouts, and logic without overcomplicating things. TypeScript works out of the box. The structure of your project mirrors the structure of your site. Everything is transparent and makes sense.

It's a tool that respects your time and gets out of your way.

### SEO benefits

Because Astro pages are pre-rendered HTML, search engine crawlers can read them immediately and completely. There's no JavaScript to execute before the content appears. This is a meaningful advantage for search rankings — search engines see exactly what your visitors see, right away.

Combined with clean HTML structure, semantic markup, and fast load times, Astro-built sites tend to perform very well in search results.

## Why I use it for every website

Some developers keep a toolkit of different frameworks and choose based on the project. I've made a deliberate choice to go deep on Astro and use it for everything I build.

The reason is quality and consistency. When you know a tool exceptionally well, you build better with it. I know exactly how to structure a project, how to optimise it, how to extend it, and how to maintain it over time. That depth benefits every client I work with.

Most of the websites I build — portfolios, business sites, blogs, landing pages — are a perfect fit for Astro. They need to be fast, reliable, well-structured, and easy to update. Astro delivers on all of that without compromise.
