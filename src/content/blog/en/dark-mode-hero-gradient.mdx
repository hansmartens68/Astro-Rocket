Astro Rocket has two distinct hero gradient classes. One covers the homepage and works across every colour scheme. The other covers all inner page heroes — but only activates for four specific colour themes. This post explains how each is built, why warm-toned themes are intentionally excluded, and how to wire up the gradient for any additional theme yourself.

## Two classes, two roles

There are two CSS classes that drive the gradient behaviour.

`.hero-dark-gradient` lives on the homepage `Hero` component and works for every colour theme in dark mode.

`.hero-page-gradient` is placed on all non-homepage heroes — the About, Blog, Projects, Contact, and Components page heroes, plus the individual blog post and project detail headers. It has an extra layer of scoping: it only activates for the blue, sky, teal, and cyan themes.

Both classes produce a `linear-gradient` from a deep brand colour at the top to pure black (`#000`) at the bottom, dark mode only.

## The homepage gradient

The homepage hero carries `.hero-dark-gradient` as a class on the `<Hero>` component:

```astro
<Hero layout="centered" size="xl" class="hero-dark-gradient" showScrollIndicator>
```

The CSS rule uses a simple `.dark` prefix, so it fires for every colour theme whenever dark mode is active:

```css
.dark .hero-dark-gradient {
  background: linear-gradient(to bottom, var(--color-brand-700) 0%, #000 100%);
}
```

The stop colour is `brand-700` — the seventh step on the active theme's brand scale. Because all twelve colour themes define their own `--brand-700` value, the gradient automatically takes on the right hue for whichever theme is selected. Switch from blue to emerald and the gradient shifts from deep indigo to deep green, with no extra CSS needed.

## The inner page gradient

Every non-homepage page hero carries `.hero-page-gradient`. For the `<Hero>` component pages this is a class prop:

```astro
<Hero layout="centered" size="sm" class="hero-page-gradient">
```

For the standalone `ArticleHero` and `ProjectHero` components it sits directly on the `<header>` element:

```html
<header class="... hero-page-gradient">
```

The CSS rule is more specific than the homepage one. Instead of scoping to `.dark`, it lists each qualifying theme explicitly:

```css
html.dark[data-theme="blue"] .hero-page-gradient,
html.dark[data-theme="sky"] .hero-page-gradient,
html.dark[data-theme="teal"] .hero-page-gradient,
html.dark[data-theme="cyan"] .hero-page-gradient {
  background: linear-gradient(to bottom, var(--color-brand-800) 0%, #000 100%);
}
```

The stop colour steps down to `brand-800` — one step darker than the homepage — giving inner pages a deeper feel that is appropriate for content rather than the primary landing experience.

## Why only cool-tone themes?

The choice to limit the gradient to four themes is a deliberate design decision, not a technical constraint.

The gradient fades from a deep brand colour to pure black. For cool-toned hues — blues, teals, and cyans — this reads as atmosphere: a dark night sky, still water, a deep studio space. The transition feels native to those palettes.

For warm or vibrant hues the same treatment produces a different result. A deep orange-to-black or lime-to-black gradient can feel heavy or mismatched against the lighter, forward energy those palettes carry. Purple and magenta present the same issue — their `brand-800` is very dark and very saturated, and a hard fade to `#000` can feel oppressive rather than atmospheric on a content page.

The eight excluded themes — orange, amber, lime, emerald, indigo, violet, purple, and magenta — are all intentionally left without the gradient so they retain their own character.

## Pages and components that carry the gradient

The complete list of locations where `hero-page-gradient` is applied:

| Location | Implementation |
|---|---|
| `/about` | `<Hero class="hero-page-gradient">` |
| `/blog` | `<Hero class="hero-page-gradient">` |
| `/projects` | `<Hero class="hero-page-gradient">` |
| `/contact` | `<Hero class="hero-page-gradient">` |
| `/components` | `<Hero class="hero-page-gradient">` |
| Individual blog posts | `<header class="... hero-page-gradient">` in `ArticleHero.astro` |
| Individual project pages | `<header class="... hero-page-gradient">` in `ProjectHero.astro` |

In every case the class is inert on excluded themes — it sits on the element doing nothing until both conditions are met: dark mode active and a qualifying theme selected.

## Adding the gradient to another theme

To extend the gradient to a theme that is currently excluded, open `src/styles/global.css` and find the `.hero-page-gradient` rule. Add a new selector line for your chosen theme:

```css
html.dark[data-theme="blue"] .hero-page-gradient,
html.dark[data-theme="sky"] .hero-page-gradient,
html.dark[data-theme="teal"] .hero-page-gradient,
html.dark[data-theme="cyan"] .hero-page-gradient,
html.dark[data-theme="emerald"] .hero-page-gradient { /* added */
  background: linear-gradient(to bottom, var(--color-brand-800) 0%, #000 100%);
}
```

The `data-theme` value must match the theme file name in `src/styles/themes/` — so `emerald` for `emerald.css`, `purple` for `purple.css`, and so on. All twelve available theme names are: `orange`, `amber`, `lime`, `emerald`, `teal`, `cyan`, `sky`, `blue`, `indigo`, `violet`, `purple`, and `magenta`.

If you want a subtler result for a warm-toned theme, adjust the stop colour. Using `brand-900` pulls the gradient darker and reduces the colour cast at the top, while `brand-700` brings more colour into the hero:

```css
html.dark[data-theme="orange"] .hero-page-gradient {
  background: linear-gradient(to bottom, var(--color-brand-900) 0%, #000 100%);
}
```

There is no rule that says warm themes cannot carry the gradient — the default exclusion is a taste preference, not a technical restriction. Experiment with `brand-900` as a starting point if you want a more conservative result for themes that don't naturally suit a strong colour-to-black transition.

## Restricting the homepage gradient too

The homepage `.hero-dark-gradient` currently activates for all twelve themes. If you want the same selective behaviour there — perhaps to disable the gradient on warm themes — apply the same technique to that rule. Replace the broad `.dark` prefix with explicit selectors:

```css
html.dark[data-theme="blue"] .hero-dark-gradient,
html.dark[data-theme="sky"] .hero-dark-gradient,
html.dark[data-theme="teal"] .hero-dark-gradient,
html.dark[data-theme="cyan"] .hero-dark-gradient {
  background: linear-gradient(to bottom, var(--color-brand-700) 0%, #000 100%);
}
```

The homepage and inner page heroes then share the same logic, just with different stop colours (`brand-700` vs `brand-800`).
