Open any page in Astro Rocket and you will notice a soft radial halo sitting behind the hero headline. It is subtle enough to feel atmospheric rather than decorative, but strong enough to anchor the brand colour immediately. This post explains exactly how it is built and gives you the two knobs you need to make it stronger, softer, or gone entirely.

## Where the glow lives

The glow is a single `<div>` rendered inside `Hero.astro`:

```astro
{/* Radial brand glow */}
<div class="bg-hero-glow pointer-events-none absolute inset-0" aria-hidden="true" />
```

It is `absolute inset-0` so it fills the full hero container, `pointer-events-none` so it never interferes with clicks, and `aria-hidden` so screen readers ignore it.

The visual work happens entirely in one CSS utility class defined in `src/styles/global.css`:

```css
.bg-hero-glow {
  background: radial-gradient(
    ellipse 80% 60% at 50% 40%,
    color-mix(in oklch, var(--color-brand-500) 15%, transparent) 0%,
    transparent 70%
  );
}
```

That is all there is to it. No JavaScript, no canvas, no filter. Just a CSS `radial-gradient`.

## Breaking down the values

```
ellipse 80% 60% at 50% 40%
```

- **`ellipse 80% 60%`** — the glow is an ellipse that is 80% of the container wide and 60% tall. A taller percentage makes it feel more "deep"; a wider one spreads it across the header.
- **`at 50% 40%`** — the centre of the ellipse sits at 50% horizontally (centred) and 40% vertically (slightly above the midpoint), which aligns it naturally with the headline text.

```
color-mix(in oklch, var(--color-brand-500) 15%, transparent)
```

- **`color-mix(in oklch, …)`** — blends the active brand colour with full transparency. Because it uses the `oklch` colour space, the mix is perceptually uniform and never produces muddy grey midpoints.
- **`var(--color-brand-500)`** — this is your theme's primary brand colour. Change the theme and the glow colour updates automatically without touching this file.
- **`15%`** — this is the opacity of the glow at its brightest centre point. At 15% it is noticeable but not harsh.

```
transparent 70%
```

The gradient fades to fully transparent at the 70% mark, leaving the outer 30% of the ellipse invisible. Moving this inward sharpens the falloff; moving it outward lets the colour bleed further.

## The orange theme override

Some brand colours need a different intensity. The orange theme, for example, sits against a warm near-white background in light mode, so 15% is too faint. Astro Rocket ships a theme-scoped override in `src/styles/themes/orange.css`:

```css
html[data-theme="orange"] .bg-hero-glow {
  background: radial-gradient(
    ellipse 80% 60% at 50% 40%,
    color-mix(in oklch, var(--color-brand-500) 28%, transparent) 0%,
    color-mix(in oklch, var(--color-brand-500) 0%, transparent) 70%
  );
}
```

The mix ratio is bumped from `15%` to `28%`. The pattern is the same — only the single percentage changes.

## How to increase the glow

To make the glow stronger, raise the mix percentage in `src/styles/global.css`:

```css
.bg-hero-glow {
  background: radial-gradient(
    ellipse 80% 60% at 50% 40%,
    color-mix(in oklch, var(--color-brand-500) 30%, transparent) 0%,  /* was 15% */
    transparent 70%
  );
}
```

Practical range: `20%`–`35%` gives a vivid but controlled effect. Above `45%` the centre starts to look washed out, especially in light mode.

You can also widen the ellipse to spread the colour across more of the hero:

```css
ellipse 100% 80% at 50% 40%   /* was 80% 60% */
```

Or push the fade point outward so the colour bleeds further:

```css
transparent 90%   /* was 70% */
```

## How to decrease the glow

Lower the mix percentage toward zero:

```css
color-mix(in oklch, var(--color-brand-500) 6%, transparent)
```

At `6%`–`8%` the glow becomes almost subliminal — you feel it more than see it. That can work very well for light-coloured themes or minimal layouts.

To remove it entirely, either delete the `<div>` from `Hero.astro` or set the mix ratio to `0%`:

```css
color-mix(in oklch, var(--color-brand-500) 0%, transparent)
```

## Per-theme tuning

The cleanest approach is to leave the default rule untouched and add an override for each theme that needs a different intensity. The pattern from `orange.css` scales to any theme:

```css
/* src/styles/themes/your-theme.css */
html[data-theme="your-theme"] .bg-hero-glow {
  background: radial-gradient(
    ellipse 80% 60% at 50% 40%,
    color-mix(in oklch, var(--color-brand-500) 22%, transparent) 0%,
    transparent 70%
  );
}
```

This keeps each theme self-contained and makes future global changes to the default safe.

## Cheat sheet

| Goal | What to change |
|---|---|
| Stronger glow | Raise mix `%` (e.g. `15%` → `30%`) |
| Softer glow | Lower mix `%` (e.g. `15%` → `7%`) |
| Remove glow | Set mix to `0%` or delete the `<div>` |
| Wider spread | Increase ellipse width (e.g. `80%` → `110%`) |
| Taller spread | Increase ellipse height (e.g. `60%` → `90%`) |
| Sharper falloff | Lower fade point (e.g. `70%` → `50%`) |
| Softer falloff | Raise fade point (e.g. `70%` → `90%`) |
| Move glow up | Lower vertical position (e.g. `40%` → `25%`) |
| Per-theme override | Add `html[data-theme="x"] .bg-hero-glow` rule |

Two lines of CSS — that is the full surface area of the hero glow. Once you know what each number controls, dialling it in to match your brand takes less than a minute.
